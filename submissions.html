Resource Submission Process

To make resources in the ABC Repository as widely accessible and extensible as possible to our community, we require all resources conform to the strict formatting guidelines outline below. For each resource you will upload a single folder with a primary text file and any supplemental media files in ato our open-source repository which is hosted on GitHub. The repository content will be rendered as a searchable website. While you may draft your resource material an any format you wish, your final submission will be a MarkDown document. Whether or not your've heard of Markdown before, please take a moment to read a recent blog post by Max Lytvyak title Markdown for Non-developers: How Writers, Educators, and Researchers Can Benefit that speaks to it's strengths as a tool to help the ABC Community connect, express, share, and evolve strategies for societal change. Learn more about Markdown's simple but powerful formatting capabilities here). For a more interactive introduction, try a short tutorial here and practice what you've learned here We understand that there may be a learning curve for most contributors who typically use other proprietary or add-funded tools like Microsoft Word or Google Docs. We do not want to constrain your creative process in any way, however you will be required to convert non-Markdown documents into a text Markdown file with appropriate formatting to submit your resource for inclusion in the ABC repository. We believe that the extra effort to make this translation into a lightweight, structured format will greatly increase the acccessibility of your contributions and increase our collective power to make change. To simply this translation process, we recommend drafting your submission directly in a Markdown editor (there are many paid and free options available). We recommend Zettlr because it is a free and open-source software with an active community of contributors and it has an academic focus, availability on most operating system (Windows, Mac, Linux, etc). Zettlr has robust documentation as well as introductory videos

A key feature of Markdown documents is the ability to store metadata about the document (call YAML Frontmatter) at the beginning of the document that can be used to provide additional content about the document in a structured manner. To ensure that all submisisons include this vital metadata, we have provided a form below that will automatically generate most of the essential Frontmatter to be included at the beginning of the Markdown file you will submit. Simply fill out the form below and copy the automatically generated code with the gray copy button, or download a Markdown file template with your metadata (and some additional styling hints) prepopulated.

Fill out the fields below and it will be immediately translated into the appropriate metadata (called "Frontmatter") for the file you will upload when you submit your resource.
